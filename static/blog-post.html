Divyanshi Rastogi | 2020/03/22

Divyanshi Rastogi is an Frontend SoftwareDeveloper at Hashout Software Technology. Born in India worshipped for its culture all over the worshipped, she has lived in Lucknow extensively but recently working in Bangalore. She loves coding,blogging and love to travel."Coding is her passion"

My first software job was building a monolithic Ruby on Rails app. Back then I hated front end development and avoided any JavaScript features. This was before React became popular. Since then I’ve realized that the ability to code up and down the stack is invaluable.

I came to React in the post-Hooks era and had trouble finding full stack tutorials using Hooks, functional components, and database persistence. I’ve written this to bridge those gaps for others. I hope it helps you.

I owe a debt of gratitude to the following articles, which were invaluable in figuring out how to fit the pieces together (although none of them did it all).

  • Create React App & Rails API Deployment to Surge & Heroku
  • Build a CRUD App in React with Hooks
  • A Rock Solid, Modern Web Stack We’re building a job board with a ReactJS

We’re building a job board with a ReactJS Hooks front end and a Rails API back end which persists data to Postgres. This app allows creating, viewing, updating and deleting jobs.

This frontend app/backend API design pattern is comprised of two separate apps in the same project, although they function as a single app when deployed.
